# User Offboarding Procedure

## Purpose
To ensure secure and timely removal of system access when a user exits the organization, reducing security risks and protecting company data.

---

## Triggers for Offboarding
- Resignation
- Contract completion
- Termination
- Role change
- Extended inactivity

---

## Offboarding Steps
1. Notification from HR or management
2. Disable user accounts
3. Revoke system access
4. Recover assigned assets
5. Transfer ownership of data
6. Archive email and files
7. Update documentation and asset register

---

## Systems Involved
- Active Directory / Azure AD
- Microsoft 365
- SharePoint
- Service Desk system
- Asset management system

---

## Security Controls
- Immediate access revocation
- MFA removal
- Data retention compliance
- Audit logging

---

## Outcome
Ensures data security, compliance, and proper asset recovery.
