# IT Asset Management

## Purpose
To ensure all IT assets are accurately tracked, securely managed, and efficiently utilized throughout their lifecycle.

This process helps reduce losses, improve accountability, and support effective capacity planning.

---

## Asset Categories
- End-user devices (laptops, desktops, monitors)
- Mobile devices (phones, tablets)
- Network equipment (routers, switches, access points)
- Software licenses
- Peripherals (keyboards, headsets, webcams)

---

## Asset Lifecycle

### 1. Procurement
- Asset request submitted
- Budget and management approval
- Vendor selection and purchase
- Asset received and inspected

### 2. Registration
- Asset tagged with unique ID
- Details recorded in asset register
- Assigned status updated

### 3. Deployment
- Device configured and secured
- User assignment documented
- Acceptance confirmed

### 4. Maintenance
- Regular health checks
- Software updates and patching
- Warranty tracking
- Repair or replacement handling

### 5. Decommissioning
- Data wiped securely
- Asset returned or disposed
- Inventory updated
- License reclaimed where applicable

---

## Asset Register Fields
- Asset ID
- Asset type
- Manufacturer & model
- Serial number
- Assigned user
- Department
- Purchase date
- Warranty status
- Asset condition
- Current status

---

## Security Controls
- Device encryption
- Endpoint protection
- Access restriction
- Secure data wiping
- Asset audit reviews

---

## Best Practices
- Perform quarterly asset audits
- Maintain clear assignment records
- Track warranties and renewals
- Enforce return process during offboarding
- Align asset management with security policies

---

## Tools Commonly Used
- Service Desk system
- Asset tracking spreadsheets or CMDB
- Endpoint management platforms
- Inventory tagging systems

---

## Outcome
Effective asset management improves:
- Cost control
- Security posture
- User accountability
- Operational efficiency
